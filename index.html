PROMPT 01번역 완료 파일 전체 한국화 다듬기

Module 3 — 강 6에서 사용 · DeepL 번역 완료 후 파일 업로드해서 한 번에 처리

첨부한 파일은 영문 PLR 전자책을 DeepL로 번역한 한국어 원고야. 이 파일 전체를 아래 기준으로 한국 독자에게 맞게 다듬어줘. 다듬기 기준: 1. 직역체 표현 → 자연스러운 한국어 구어체로 교체 2. '당신은 ~할 수 있습니다' → '~하시면 됩니다' 식으로 수정 3. 미국식 단위(파운드, 온스, 화씨, 컵) → 한국식(kg, g, ℃, ml)으로 변환 4. 원문 의미와 챕터 구성은 그대로 유지 수정된 전체 원고를 완성본으로 출력해줘.

💡 파일을 첨부한 후 이 프롬프트를 함께 입력하세요. Word(.docx) 또는 PDF 파일 모두 가능합니다.

PROMPT 02번역 파일 식재료 & 문화 현지화

건강·요리·라이프스타일 관련 PLR에서 사용 · 파일 업로드 후 한 번에 처리

첨부한 파일은 영문 PLR 전자책을 번역한 한국어 원고야. 이 파일 전체에서 한국 독자에게 맞지 않는 표현을 찾아서 전부 바꿔줘. 변환 기준: - 파운드(lb) → kg (1lb = 0.45kg) - 온스(oz) → g (1oz = 28g) - 화씨(°F) → 섭씨(°C) - 컵(cup) → ml (1cup = 240ml) 식재료 변환: - 아보카도 오일 → 들기름 또는 참기름 - 케일 → 깻잎 또는 시금치 - 퀴노아 → 현미 또는 귀리 - 버터밀크 → 플레인 요거트 - 그래놀라 → 오트밀 수정된 전체 원고를 완성본으로 출력해줘.

💡 변환 목록은 PLR 주제에 따라 직접 추가·수정해서 사용하세요.

⚠️ 결과물은 반드시 직접 검수하세요.

PROMPT 03번역 파일 전체 품질 검수 & 개선 제안

번역 다듬기가 끝난 후 최종 품질 확인 단계에서 사용

첨부한 파일은 PLR 전자책을 번역하고 1차 다듬기까지 완료한 한국어 원고야. 이 파일 전체를 검토하고 아래 항목을 점검해줘. 검토 항목: 1. 아직 어색하게 남아있는 직역체 표현 목록 2. 한국 독자에게 낯설 수 있는 표현이나 사례 3. 문맥상 의미가 불명확한 문장 4. 챕터 간 흐름이 부자연스러운 부분 각 항목별로 문제가 되는 원문과 개선 제안을 함께 정리해줘. 수정이 필요 없는 부분은 '이상 없음'으로 표시해줘.

💡 수정 제안을 받은 후 수락할 부분만 골라서 PROMPT 01로 재처리하면 깔끔합니다.

PROMPT 04번역 파일 목차 & 챕터 구성 정리

번역이 끝난 후 목차를 뽑아 상품 소개글에 활용할 때 사용

첨부한 파일은 PLR 전자책을 번역한 한국어 원고야. 이 파일 전체를 읽고 아래 내용을 정리해줘. 정리 항목: 1. 전체 목차 (챕터 제목 + 소제목 포함) 2. 각 챕터 핵심 내용 1~2문장 요약 3. 챕터 간 흐름이 어색한 부분이 있으면 개선 방향 제안 4. 판매 소개글에 쓸 수 있는 목차 형태로 정리 출력 형태는 상품 소개글에 바로 복사해서 쓸 수 있도록 깔끔하게 정리해줘.

💡 정리된 목차를 리틀리 상품 소개글 미리보기 섹션에 그대로 붙여넣으세요.

PROMPT 04-A 번역 원고 퀄리티 업그레이드 — 내용 심화

PLR 원고 번역·다듬기 완료 후 · 내용이 너무 얕거나 단순할 때 깊이와 밀도를 높이는 단계

너는 논픽션 전자책 편집 전문가야. 첨부한 파일은 PLR 전자책을 번역·다듬기까지 완료한 한국어 원고야. 원문의 구조와 흐름은 그대로 유지하면서, 아래 기준으로 내용의 깊이와 퀄리티를 끌어올려줘. ━━━━━━━━━━━━━━━━━━━━ 📄 업그레이드 기준 ━━━━━━━━━━━━━━━━━━━━ 1. 내용 심화 - 각 챕터에서 핵심 주장의 근거나 이유를 1~2가지 추가 - '~하세요'로 끝나는 조언에는 '왜 그래야 하는지' 설명을 덧붙이기 - 너무 추상적인 설명은 구체적인 예시나 수치로 보완 2. 실용성 강화 - 독자가 바로 실천할 수 있는 '액션 포인트'를 각 챕터 끝에 1~3가지 추가 - '이렇게 해보세요' 형식의 단계별 가이드로 전환 가능한 부분은 전환 3. 가독성 향상 - 너무 긴 단락(6줄 이상)은 자연스럽게 나누기 - 중요한 문장은 앞에 '💡' 또는 '→' 같은 시각 요소로 강조 - 챕터 시작 부분에 '이번 챕터에서 배울 것' 한 줄 추가 4. 톤 유지 - 원문의 문체와 말투는 그대로 유지 - 추가되는 내용이 기존 원고와 어색하지 않게 자연스럽게 녹아들도록 수정된 전체 원고를 완성본으로 출력해줘.

💡 챕터 수가 많으면 '1챕터씩 업그레이드해줘'라고 나눠서 요청하는 게 결과물 품질이 더 높습니다.

⚠️ 추가된 내용이 원래 원고와 자연스럽게 이어지는지 반드시 직접 읽어보며 확인하세요.

PROMPT 04-B 번역 원고 퀄리티 업그레이드 — 문장·구성 리라이팅

내용은 괜찮지만 문장이 딱딱하거나 전체 흐름이 어색할 때 · 완성도를 한 단계 끌어올리는 단계

너는 한국어 논픽션 편집 전문가야. 첨부한 파일은 PLR 전자책을 번역·다듬기까지 완료한 한국어 원고야. 내용과 정보는 유지하면서, 아래 기준으로 문장과 전체 구성을 더 읽히는 글로 리라이팅해줘. ━━━━━━━━━━━━━━━━━━━━ 📄 리라이팅 기준 ━━━━━━━━━━━━━━━━━━━━ 1. 문장 리라이팅 - 번역투 냄새가 남은 문장은 한국어 원고처럼 자연스럽게 재작성 - 수동태·명사형 문장 → 능동태·서술형으로 전환 (예: '~이 이루어집니다' → '~할 수 있어요') - 한 문장에 내용이 2개 이상 섞인 경우 → 두 문장으로 분리 - 딱딱한 연결어('또한', '그러므로') → 자연스러운 표현으로 교체 (예: '그러니까', '그래서', '그런 만큼') 2. 챕터 구성 강화 - 각 챕터 첫 문단: 독자의 공감을 이끌어내는 문장으로 시작 - 각 챕터 마지막 문단: 다음 챕터로 자연스럽게 이어지는 브릿지 문장 추가 - 내용 나열 방식이 단조로우면 '스토리 → 정보 → 실천' 구조로 재배치 3. 전체 톤 통일 - 챕터마다 말투가 달라지는 부분 통일 - 선택한 말투: [친근한 블로그체 / 신뢰감 있는 전문가체] ← 하나 선택해서 입력 수정된 전체 원고를 완성본으로 출력해줘.

💡 04-A(내용 심화)와 04-B(문장 리라이팅)를 순서대로 모두 거치면 가장 높은 완성도를 얻을 수 있어요.

⚠️ 전체 리라이팅 후에는 PROMPT 03(품질 검수)으로 최종 점검하는 것을 권장합니다.

PROMPT 05상품명 5가지 생성

Module 3 — 강 8 · [대상] + [기간/방법] + [구체적 결과] 공식 적용

아래 전자책 내용을 바탕으로 잘 팔리는 상품명을 5가지 만들어줘. 상품명 작성 기준: - 형식: [대상] + [기간 또는 방법] + [구체적 결과] - 숫자나 기간을 반드시 포함해줘 (예: 30일, 7가지, 한 달) - 한국 독자에게 친숙한 표현 사용 - 20자 내외로 간결하게 - '~하는 법', '~가이드', '~완전정복' 같은 실용적 키워드 포함 전자책 주제: [주제 입력] 핵심 내용: [핵심 내용 2~3줄 입력]

💡 5가지 중 가장 마음에 드는 것을 고르고, 조합해서 사용해도 좋습니다.

PROMPT 06PAPA 공식 소개글 작성

Problem → Agitate → Promise → Action 구조로 구매욕을 자극하는 소개글

아래 정보를 바탕으로 전자책 소개글을 PAPA 공식으로 작성해줘. PAPA 구조: 1. Problem(문제 제기): 독자가 공감할 수 있는 고민이나 문제 상황 2. Agitate(공감과 자극): 그 문제가 왜 생기는지, 혼자서는 해결하기 어려운 이유 3. Promise(해결책 제시): 이 전자책이 어떻게 해결해주는지 4. Action(구매 유도): 지금 바로 시작해야 하는 이유 각 단계를 3~5문장으로 작성해줘. 문체는 존댓말이지만 친근한 톤으로. 전자책 제목: [제목 입력] 핵심 타깃: [타깃 독자 입력] 주요 내용: [전자책 주요 내용 3~5줄 입력]

💡 작성된 소개글을 리틀리 상품 페이지에 바로 붙여넣으세요.

PROMPT 07소개글 톤 & 어투 개선

소개글 초안 작성 후 마무리 다듬기용

아래 전자책 소개글을 더 자연스럽고 설득력 있게 다듬어줘. 개선 기준: - 너무 딱딱하거나 광고스러운 표현은 부드럽게 - '독자가 사야 하는 이유'가 명확하게 드러나도록 - 문장이 너무 길면 짧게 끊어줘 - 첫 문장은 반드시 독자의 공감을 이끌어내는 문장으로 - 마지막 문장은 행동을 유도하는 문장으로 [여기에 소개글 초안을 붙여넣기]

💡 PROMPT 06으로 초안을 만든 후 이 프롬프트로 마무리하면 완성도가 높아집니다.

PROMPT 08독자 후기 요청 문구 작성

구매자에게 후기를 요청할 때 사용하는 메시지

전자책 구매자에게 보낼 후기 요청 메시지를 작성해줘. 조건: - 강요하지 않고 자연스럽게 부탁하는 톤 - 전자책이 도움이 됐는지 먼저 확인하는 구조 - 후기 남겨주면 좋은 이유를 한 줄로 설명 - 카카오톡 또는 이메일로 발송할 수 있도록 간결하게 - 3버전(짧은 버전 / 중간 버전 / 정중한 버전)으로 작성해줘 전자책 제목: [제목 입력]

💡 초기 구매자 5~10명에게 발송해서 첫 리뷰를 빠르게 모으세요.

PROMPT 09인스타그램 정보성 카드뉴스 텍스트 작성

Module 6 — 강 15 · 저장율·공유율이 높은 콘텐츠 포맷

인스타그램 카드뉴스용 텍스트를 작성해줘. 조건: - 총 7~8장 구성 (1장: 훅 / 2~6장: 본문 / 7장: 요약 / 8장: CTA) - 각 카드는 2~3줄 이내로 짧게 - 첫 장(훅)은 스크롤을 멈추게 하는 강렬한 문장으로 - 마지막 장(CTA)에는 '댓글에 [키워드] 달아주세요'로 마무리 - 이모지 적절히 활용 - 문체: 친근한 반말 또는 존댓말 주제: [예: 직장인이 부업 시작하기 전 알아야 할 5가지] 핵심 내용: [내용 입력]

💡 마지막 카드 CTA 키워드가 DM 자동발송과 연동되도록 설정하세요.

PROMPT 10스레드 바이럴 글 작성

Module 6 — 강 15 · 신규 계정도 노출되는 스레드 특성 활용

스레드에 올릴 바이럴 글을 3가지 버전으로 작성해줘. 조건: - 전체 3~5줄로 짧게 - 첫 문장은 호기심을 자극하는 문장으로 (스크롤 멈추게 하기) - '대부분이 모르는', '아직 알려지지 않은', '이거 알면 생각 바뀜' 등 활용 - 마지막 문장은 반드시 '궁금하면 댓글 달아줘' 또는 '원하는 분 댓글 주세요'로 - 스마트스토어, 유튜브, 블로그가 아닌 새로운 방법임을 암시 - 해시태그는 포함하지 말 것 (자연스러운 글처럼 보이도록) 홍보할 내용: [전자책 주제 또는 핵심 메시지 입력]

💡 3가지 버전을 요일별로 나눠서 올리면 어떤 게 반응이 좋은지 테스트할 수 있어요.

🎁 CHAPTER 06

무료 전자책 제작 프롬프트

🎯 무료 전자책이란?

유료 전자책을 홍보하기 위해 무료로 배포하는 리드 마그넷(Lead Magnet) 콘텐츠입니다.
독자에게 먼저 가치를 주고 신뢰를 쌓은 뒤, 유료 전자책으로 자연스럽게 연결하는 것이 핵심입니다.
5~10페이지 분량으로 제작하며, PDF · DOCX · HTML 형식 중 원하는 형태로 출력할 수 있습니다.

📁 출력 형식별 활용 가이드

PDF Canva · Adobe Express로 디자인 후 PDF 저장 → 이메일 첨부, 카카오채널 배포에 최적
DOCX Word로 바로 편집 가능 → 내용 수정이 많을 때 유리, 이후 PDF로 변환해서 배포
HTML 웹 페이지형 전자책 → 링크 하나로 공유 가능, 스마트폰에서 바로 읽기 편함

📚 합본 제작이란?

같은 주제·카테고리의 PLR 전자책 2~3권을 하나의 고퀄리티 전자책으로 통합하는 작업입니다.
단순히 내용을 붙이는 게 아니라, 중복을 제거하고 흐름을 재설계해서 페이지 수와 완성도를 동시에 높이는 것이 핵심입니다.
소싱 비용 대비 상품 가치를 극대화할 수 있는 전략입니다.

PROMPT 25합본 전 전자책 2권 비교 분석

합치기 전에 먼저 실행 · 두 전자책의 겹치는 내용·차이점·통합 가능성을 파악하는 사전 분석 단계

너는 전자책 편집 전문가야. 아래 두 전자책의 목차와 핵심 내용을 비교 분석해서 합본 제작 전 전략을 세워줘. ━━━━━━━━━━━━━━━━━━━━ ✏️ 입력 정보 ━━━━━━━━━━━━━━━━━━━━ [전자책 A] 제목: [입력] [전자책 A] 목차 또는 주요 내용: [여기에 붙여넣기] [전자책 B] 제목: [입력] [전자책 B] 목차 또는 주요 내용: [여기에 붙여넣기] ━━━━━━━━━━━━━━━━━━━━ 📄 분석 항목 ━━━━━━━━━━━━━━━━━━━━ 1. 두 전자책의 공통 주제·겹치는 챕터 목록 - 완전히 같은 내용 / 비슷하지만 관점이 다른 내용 구분 2. 각 전자책의 고유한 내용 (상대방에 없는 부분) - A에만 있는 내용 - B에만 있는 내용 3. 합본 시 통합 전략 제안 - 어떤 챕터를 기준으로 삼을지 - 중복 내용 처리 방향 (삭제 / 보완·확장 / 다른 관점으로 재구성) - 합본 후 예상 완성도 평가: 상 / 중 / 하 + 이유 4. 합본 후 새 목차 초안 (제안) - 두 전자책의 장점만 살린 재구성 목차

💡 목차 전체를 붙여넣기 어려우면 각 전자책의 챕터 제목과 소제목만 입력해도 충분합니다.

🔁 반복 활용 팁: 합칠 전자책만 바꾸면 어떤 조합이든 사전 분석을 빠르게 뽑을 수 있어요.

PROMPT 26중복 제거 & 자연스러운 합본 원고 완성

PROMPT 25 분석 후 실행 · 두 전자책을 하나의 자연스러운 원고로 통합하는 핵심 단계

너는 전문 편집자야. 아래 두 전자책 원고를 하나의 완성된 전자책으로 자연스럽게 합쳐줘. 단순히 붙이는 게 아니라, 진짜 처음부터 기획된 책처럼 만들어야 해. ━━━━━━━━━━━━━━━━━━━━ ✏️ 입력 정보 ━━━━━━━━━━━━━━━━━━━━ 합본 후 전자책 제목 (가제): [입력 또는 '제안해줘'로 입력] PROMPT 25에서 나온 새 목차: [붙여넣기] [전자책 A] 원고: [붙여넣기] [전자책 B] 원고: [붙여넣기] ━━━━━━━━━━━━━━━━━━━━ 📄 합본 작업 기준 ━━━━━━━━━━━━━━━━━━━━ 1. 중복 내용 처리 - 완전히 같은 내용 → 더 잘 쓰인 쪽 채택 또는 두 관점을 하나로 통합 - 비슷하지만 관점이 다른 내용 → '관점 A / 관점 B' 형식으로 보완해서 오히려 풍부하게 2. 흐름 재설계 - 챕터 간 전환 문장을 새로 작성해서 끊기는 느낌 없애기 - 앞 챕터와 뒤 챕터가 자연스럽게 이어지도록 브릿지 문단 추가 3. 톤 통일 - 두 원고의 문체가 다를 경우 → 더 자연스러운 쪽 기준으로 전체 톤 통일 - 직역체나 어색한 표현은 자연스러운 한국어로 함께 다듬기 4. 구성 강화 - 각 챕터 시작에 '이번 챕터에서 다룰 것' 한 줄 추가 - 각 챕터 끝에 '핵심 요약 1~2줄' 추가 - 전체 마지막에 '총정리' 또는 '실천 체크리스트' 섹션 추가 완성된 합본 원고 전체를 출력해줘.

🔁 반복 활용 팁: 합칠 원고와 목차만 바꾸면 어떤 조합의 합본도 같은 방식으로 처리할 수 있어요.

PROMPT 27합본 완성 원고 최종 품질 검토

합본 작업이 끝난 후 마지막으로 실행 · 전체 흐름·중복·어색함을 최종 점검하는 단계

너는 꼼꼼한 전자책 편집장이야. 아래 합본 완성 원고를 처음부터 끝까지 읽고 최종 품질 검토 보고서를 작성해줘. ━━━━━━━━━━━━━━━━━━━━ ✏️ 입력 정보 ━━━━━━━━━━━━━━━━━━━━ 합본 전자책 제목: [입력] 합본 완성 원고: [붙여넣기] ━━━━━━━━━━━━━━━━━━━━ 📄 검토 항목 ━━━━━━━━━━━━━━━━━━━━ 1. 잔여 중복 점검 - 아직 겹치는 내용이나 비슷한 문장이 남아 있는 부분 목록 - 위치(챕터명)와 개선 방향 함께 제시 2. 흐름 부자연스러운 구간 - 챕터 간 전환이 어색한 부분 - 앞뒤 맥락이 맞지 않는 문단 3. 톤 불일치 구간 - 문체가 갑자기 달라지거나 어색해진 부분 - 수정 제안 문장 함께 제시 4. 내용 공백 또는 약한 챕터 - 다른 챕터에 비해 내용이 빈약한 부분 - 보강할 수 있는 방향 제안 5. 종합 완성도 평가 - 항목별 평가: 흐름 / 중복 제거 / 톤 통일 / 내용 충실도 - 별점 표시 (★ 5개 만점) - 판매 전 추가 수정이 꼭 필요한 부분 우선순위 3가지 수정이 필요 없는 항목은 '이상 없음'으로 표시해줘.

⚠️ 합본 작업 전 반드시 확인하세요

같은 카테고리라도 방향이 다른 책은 합쳐도 어색할 수 있습니다.

예를 들어 '다이어트'라는 큰 카테고리 안에서도 '식단 중심 다이어트'와 '운동 루틴 중심 다이어트'처럼 접근 방식이 완전히 다른 경우, AI가 아무리 잘 합쳐도 전체 맥락이 튀거나 독자 입장에서 혼란스러울 수 있어요.

✅ 잘 합쳐지는 조합

· 같은 방법론, 다른 깊이 (기초편 + 심화편)
· 같은 목표, 서로 보완되는 접근법
· 비슷한 독자를 대상으로 한 내용

❌ 합본 시 주의가 필요한 조합

· 접근 방식이나 철학이 정반대인 내용
· 타깃 독자층이 다른 전자책
· 난이도 차이가 너무 큰 경우 (초급 + 전문가)

합본이 완성되면 반드시 처음부터 직접 한 번 통독해보세요.
AI 검토(PROMPT 27)는 1차 필터 역할이고, 최종 판단은 직접 읽어봐야만 알 수 있습니다. 독자 입장에서 읽었을 때 자연스럽게 이어지는지 확인하는 것이 가장 확실한 품질 검증입니다.
